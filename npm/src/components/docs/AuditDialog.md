# AuditDialog.vue

Компонент відображає журнал аудиту, наданий зовнішнім агентом. Він дозволяє переглядати записи, розширювати їх для детального огляду та взаємодіяти з ними (наприклад, відповідати або затверджувати).

## Поведінка

1. При ініціалізації компонента, якщо не вказано інше, журнал аудиту не завантажується.
2. При натисканні кнопки "Оновити" у заголовку, компонент ініціює завантаження списку записів журналу.
3. Під час завантаження, відображається індикатор завантаження.
4. Якщо список записів порожній і завантаження завершено, відображається повідомлення "No requests yet".
5. Кожен запис журналу відображається як рядок.
6. Натискання на заголовок рядка розкриває або приховує деталі цього запису.
7. Якщо запис має статус "needs_approval" та потребує затвердження, відображаються кнопки "Підтвердити" та "Відхилити".
8. При натисканні на "Підтвердити" або "Відхилити", компонент викликає відповідну дію агента, оновлює журнал і надсилає подію `changed`.
9. При взаємодії з `RequestView` (наприклад, введення відповіді), компонент викликає дію агента `respond`, оновлює журнал і надсилає подію `changed`.
10. При зміні властивості `modelValue` зовнішнім компонентом, компонент оновлює свій стан відображення.

## Гарантії поведінки

* Компонент завжди відображає актуальний стан журналу після успішного виклику `refresh`.
* Під час виконання операцій (відповідь, затвердження) компонент блокує взаємодію з відповідним записом, щоб уникнути подвійних викликів.
* У разі помилки при завантаженні або взаємодії з агентом, користувачеві відображається сповіщення.
* Деталі запису відображаються лише тоді, коли відповідний запис розгорнутий.
