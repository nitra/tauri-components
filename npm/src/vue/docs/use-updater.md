# use-updater.js

## Огляд

Модуль забезпечує перевірку наявності новіших версій застосунку та ініціює процес оновлення. Публічна функція `useUpdater` виконує цю логіку, забезпечуючи перехоплення помилок (fail-safe) та запобігаючи виняткам назовні. Логіка роботи залежить від конфігураційних файлів: tauri.conf.json та default.json.

## Поведінка

1. При виклику `useUpdater` у середовищі розробки (dev builds) функціонал оновлення не виконується.
2. При монтуванні компонента, через `FIRST_CHECK_DELAY_MS` часу, викликається перевірка на наявність оновлення.
3. Після першої перевірки, функція `useUpdater` запускає періодичну перевірку на наявність оновлення кожні `CHECK_INTERVAL_MS` часу.
4. При виклику перевірки на оновлення, якщо виявлено доступну новішу версію, відображається діалогове вікно з пропозицією встановити оновлення.
5. Якщо користувач підтверджує встановлення, запускається процес завантаження та встановлення оновлення. Під час завантаження відображається прогрес.
6. Після успішного встановлення оновлення, відображається діалогове вікно з пропозицією негайного перезапуску.
7. Якщо користувач підтверджує перезапуск, виконується перезапуск застосунку.
8. У разі будь-якої помилки під час перевірки або встановлення, користувач не отримує винятків, а замість цього відображається відповідне повідомлення або логується помилка.
9. При демонтуванні компонента, періодичні таймери перериваються.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
