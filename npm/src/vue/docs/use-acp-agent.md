## Огляд

Модуль ініціалізує шлюз агента та налаштовує доменне мостове з'єднання MCP, якщо каталог доступний. Він звертається до мережі для відправки запитів до агента. Публічна функція `useAcpAgent` використовується для передачі намірів агенту, що дозволяє оновлювати рівень моделі. Після цього модуль надає відповідь та схвалює запит. Цей модуль є Read-only і не виконує записів у файлову систему чи бази даних.

## Поведінка

1. Викликати `useAcpAgent` для ініціалізації шлюзу агента в додатку.
2. Викликати `loadEnv` для зчитування за замовчуванням агента на машині та запуску доменного мостового з'єднання MCP, якщо каталог доступний.
3. Викликати `request` для надсилання запиту з наміром.
4. При виклику `request`, якщо в опціях вказано `modelTier`, оновити поточний рівень моделі.
5. Викликати `request` для надсилання запиту з наміром, використовуючи поточний рівень моделі за замовчуванням, якщо він не був перевизначений.
6. Викликати `respond` для надання відповіді на запит, використовуючи ідентифікатор запиту та повідомлення.
7. Викликати `approve` для схвалення запиту, використовуючи ідентифікатор запиту та стан схвалення.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
