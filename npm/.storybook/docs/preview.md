## Огляд

Цей файл піднімає Storybook у Quasar-контексті з URL /storybook, щоб сторінки документації працювали в тому ж середовищі, що й застосунок, і мали доступ до базових Quasar-компонентів та потрібного директивного API. Він також підключає підтримку self-hosted SVG-іконок через app.use, щоб приклади рендерилися з локальними ресурсами.

## Поведінка

1. Піднімає Storybook у Quasar-контексті й робить доступними всі базові Quasar-компоненти та потрібний директивний API для сторінок документації.
2. Додає підтримку локально зібраних Material Symbols і переводить внутрішні іконки Quasar на self-hosted варіант без залежності від Google Fonts CDN.
3. Якщо в компоненті використано іконку з префіксом для Material Symbols, підбирає відповідний SVG-варіант і віддає його напряму для рендерингу.
4. Якщо іконка не належить до підтримуваних варіантів, залишає стандартну поведінку Quasar без втручання.
5. Увімкнює сповіщення Quasar та вимикає dark-режим для стабільного вигляду сторінок Storybook.
6. Налаштовує Storybook controls для зручнішого керування кольорами й датами в прикладах.
