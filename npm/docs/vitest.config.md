## Огляд

Файл визначає та збирає тестові файли, виключаючи директорії `node_modules`, `dist` та звіти Stryker. Тести виконуються у середовищі `node`, ізолюючи кожен тест у окремому процесі для запобігання взаємному впливу фікстур (test.mdc). Після виконання тестів збираються дані про покриття коду, використовуючи провайдер `v8` та генеруючи звіт у форматах `lcov` та `text-summary`.

## Поведінка

1. Визначає, які файли містять тести, включаючи файли з конвенцією `test` у піддиректоріях та інтеграційні набори в кореневій директорії `tests`.
2. Ігнорує директорії `node_modules`, скомпільовані файли `dist` та тимчасові звіти Stryker.
3. Виконує тести у середовищі `node`.
4. Ізолює виконання тестів, запускаючи кожен тест у окремому процесі (`forks`), щоб запобігти взаємному впливу між тестовими фікстурами (test.mdc).
5. Збирає звіт про покриття коду, використовуючи провайдер `v8` та генераючи звіти у форматах `lcov` та `text-summary`.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `node_modules`.
