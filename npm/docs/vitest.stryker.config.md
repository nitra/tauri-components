## Огляд

Файл задає окремий прогін mutation testing для JS-частини Storybook-проєкту, щоб перевіряти unit-тести без залежності від Storybook-конфігурацій. Скоуп обмежено тестовими файлами в workspace, без `node_modules`. Для результатів формуються звіти у форматах HTML і JSON, щоб їх можна було переглядати й аналізувати окремо від основного запуску.

## Поведінка

1. Використовує окрему `vitest`-конфігурацію для mutation testing, щоб ізолювати перевірку JS-специфіки від браузерного `storybook`-проєкту.
2. Запускає лише unit-тести у Node-оточенні, зосереджуючись на стабільній перевірці коду без впливу Storybook-налаштувань.
3. Відбирає лише тестові файли з робочої області проєкту, а `node_modules` свідомо пропускає.
4. Не залучає артефакти збірки та звіти mutation testing до виконання, щоб не змішувати результати з тимчасовими файлами.
5. Формує звіти покриття у форматах для перегляду результатів і підсумкової оцінки якості тестів.

## Гарантії поведінки

- Свідомо пропускає шляхи: `node_modules`.
