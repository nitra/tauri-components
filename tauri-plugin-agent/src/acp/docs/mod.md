# mod.rs

## Огляд

Цей модуль керує взаємодією з зовнішніми кодувальними агентами, запускаючи їх як підпроцеси через протокол `agent_client_protocol`. Він керує повним життєвим циклом сесії: від ініціалізації до обробки команд у рамках сесії. Сесія залишається живою між кількома промптами, оскільки `acp_spawn_agent` запускає фонове завдання, яке володіє з'єднанням з агентом та обробляє команди. Сповіщення про оновлення сесії реемітються як `acp://session-update`, а запити на дозвіл (`session/request_permission`) тримаються відкритими, зберігаючи відповідник у `AcpState` до моменту, коли веб-інтерфейс викликає `acp_respond_permission`. Модуль також надає механізми для конфігурування агентів через `AcpConfig` та `acp_config`.

## Поведінка

AcpState зберігає ідентифікатори активних сесій та відповідники для очікуваних запитів на дозвіл.
SpawnAgentArgs описує конфігурацію для запуску зовнішнього агента.
acp_spawn_agent запускає зовнішнього агента як підпроцес, ініціалізує сесію та повертає внутрішній ключ сесії.
acp_prompt надсилає запит на генерацію відповіді в існуючу сесію та повертає кінцеву причину зупинки.
acp_cancel відсилає команду на припинення поточного запиту в активній сесії.
acp_respond_permission розв'язує очікуваний запит на дозвіл, який зробив агент, використовуючи наданий ідентифікатор та опцію.
AcpConfig містить конфігурацію, що визначає стандартний тип агента.
acp_config повертає конфігурацію, зчитуючи стандартний тип агента з системної змінної середовища.

## Публічний API

AcpState — Зберігає стан для кожної запущеної сесії та відгуків на запити дозволу.
SpawnAgentArgs — Визначає аргументи для запуску агента.
acp_spawn_agent — Запускає підпроцес агента, ініціалізує сесію та повертає внутрішній ключ сесії.
acp_prompt — Надсилає запит агенту в активній сесії та очікує завершення цього етапу.
acp_cancel — Змушує агента припинити поточний запит.
acp_respond_permission — Завершує очікування на запит дозволу, ініційований агентом.
AcpConfig — Встановлює за замовчуванням тип агента для машини, виходячи з системної змінної.
acp_config — Завантажує конфігурацію за замовчуванням для агента.

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
