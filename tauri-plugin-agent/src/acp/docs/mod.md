# mod.rs

## Огляд

Цей модуль керує взаємодією з зовнішніми кодовими агентами, запускаючи їх як підпроцеси через протокол `agent_client_protocol`. Функція `acp_spawn_agent` ініціює фонову задачу, яка володіє з'єднанням з агентом та обробляє команди, відправлені через mpsc канал. Модуль керує життєвим циклом сесії, включаючи ініціалізацію, створення нових сесій та обробку запитів. Відповіді на запити від агента, такі як `session/update`, пересилаються у вебв'ю як `acp://session-update`. Запити дозволів від агента (`session/request_permission`) пересилаються як `acp://permission-request` і зберігаються в `AcpState` до моменту, коли вебв'ю надає відповідь через `acp_respond_permission`. Модуль дозволяє відправляти запити (`acp_prompt`) та скасовувати їх (`acp_cancel`).

## Поведінка

AcpState зберігає активні сесії та очікувачі відповідей на запити дозволів.
SpawnAgentArgs описує конфігурацію для запуску агента.
acp_spawn_agent запускає зовнішнього агента як підпроцес, ініціалізує сесію та повертає внутрішній ключ сесії.
acp_prompt відправляє запит на генерацію відповіді в активну сесію та очікує на фінальний стан цього виклику.
acp_cancel відправляє команду на припинення поточного запиту в активну сесію.
acp_respond_permission розв'язує відкладений запит дозволу, який зробив агент, надаючи обраний варіант.
AcpConfig містить конфігурацію, зокрема тип агента за замовчуванням.
acp_config повертає конфігурацію агента, зчитуючи значення з системної змінної середовища.

## Публічний API

AcpState — Зберігає стан для кожної запущеної сесії та відгуків на запити дозволу.
SpawnAgentArgs — Визначає аргументи для запуску агента.
acp_spawn_agent — Запускає підпроцес агента, ініціалізує сесію та повертає внутрішній ключ сесії.
acp_prompt — Надсилає запит агенту в активній сесії та очікує завершення ходу.
acp_cancel — Просить агента припинити поточний запит.
acp_respond_permission — Завершує очікування на запит дозволу, зроблений агентом.
AcpConfig — Вказує тип агента за замовчуванням для машини, на основі встановлених CLI.
acp_config — Налаштовує конфігурацію агента.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
