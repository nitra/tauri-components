# mod.rs

## Огляд

Модуль керує взаємодією з зовнішніми кодувальними агентами, запускаючи їх як підпроцеси через протокол `agent_client_protocol`. Він ініціалізує, керує та підтримує життєвий цикл сесії. Запуск агента через `acp_spawn_agent` створює фонову задачу, яка володіє `ConnectionTo<Agent>` та обробляє команди, надсилані через mpsc-канал, забезпечуючи підтримку сесії між послідовними промптами. Модуль обробляє повідомлення `session/update` та перетворює їх на `acp://session-update` для веб-інтерфейсу. Він також керує запитами на дозвіл (`session/request_permission`), які перетворюються на `acp://permission-request` та зберігаються до моменту відповіді від веб-інтерфейсу.

## Поведінка

AcpState зберігає активні сесії та очікувачі відповідей на запити дозволів.
SpawnAgentArgs описує конфігурацію для запуску агента.
acp_spawn_agent запускає зовнішнього агента як підпроцес, ініціалізує сесію та повертає внутрішній ключ сесії.
acp_prompt надсилає запит на генерацію відповіді в активну сесію та очікує на завершення цього туру.
acp_cancel відсилає команду на скасування поточного запиту в активну сесію.
acp_respond_permission вирішує відкладений запит дозволу, який був зроблений агентом.
AcpConfig містить конфігурацію за замовчуванням для агента.
acp_config отримує конфігурацію за замовчуванням для агента з системних змінних.

## Публічний API

AcpState — Зберігає стан кожного запущеного сеансу, а також очікувачі на відповідь на запити дозволу.
SpawnAgentArgs — Визначає аргументи для запуску агента.
acp_spawn_agent — Запускає підпроцес агента з заданими аргументами, ініціалізує сеанс та підтримує його в фоновому режимі, повертаючи внутрішній ключ сеансу.
acp_prompt — Надсилає запит агенту в існуючому сеансі та чекає завершення цього кроку, повертаючи лише причину зупинки.
acp_cancel — Просить агента припинити поточний запит, що перебуває в обробці.
acp_respond_permission — Завершує очікування на відповідь на запит дозволу, ініційований агентом.
AcpConfig — Встановлює за замовчуванням тип агента для машини, виходячи з системної змінної, що вказує на встановлений CLI.
acp_config — Налаштовує конфігурацію агента.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
