# lib.rs

## Огляд

Цей файл надає реалізацію плагіна агента. Він реєструє команди для журналу запитів (`journal_*`) та конфігурації `omlx_config`. Для використання плагіна необхідно зареєструвати його в конструкторі застосунку за допомогою `tauri::Builder::default.plugin` та надати дозвіл `agent:default` у файлі можливостей застосунку.

## Поведінка

1. Викликати `init` для ініціалізації плагіна агента.
2. Плагін реєструє команди для журналу запитів (`journal_*`) та конфігурації `omlx_config`.
3. Плагін реєструє команди для клієнта ACP (`acp_spawn_agent`, `acp_prompt`, `acp_cancel`, `acp_respond_permission`, `acp_config`).
4. Плагін реєструє команди для мостового зв'язку MCP (`acp_register_catalog`, `acp_mcp_tool_result`, `acp_start_mcp_bridge`).
5. Під час налаштування застосунку плагін створює та керує станом ACP (`AcpState`).
6. Під час налаштування застосунку плагін створює та керує станом мосту MCP (`McpBridgeState`).

## Публічний API

init — Ініціалізує плагін агента, налаштовуючи команди для журналювання та omlx, клієнтські команди ACP (запуск, запит, скасування зовнішнього кодувального агента та обробка його запитів на дозвіл), а також мост до домену MCP (виставляє каталог інструментів програми для будь-якого запущеного агента ACP).
