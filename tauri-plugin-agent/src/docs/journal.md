# journal.rs

## Огляд

Модуль є сховищем журналів для шлюзу агента, де кожен запит зберігається як JSON-об'єкт у відповідній директорії. Усі файлові операції виконуються через Rust. Записи зберігаються як JSON-об'єкти, причому повідомлення та дії залишаються без схеми. Модуль дозволяє створювати нові записи зі статусом `pending`, завантажувати їх за ID, оновлювати існуючі записи за допомогою патчів (зміна статусу чи полів) та отримувати список усіх записів у директорії, відсортований за часом створення. Модуль є fail-safe: він перехоплює помилки та не кидає винятків назовні, повертаючи порожнє значення при певних помилках. Директорія запитів визначається шаром команд, що забезпечує ізоляцію журналів для кожного споживача програми.

## Поведінка

create_in створює новий запис зі статусом `pending` у вказаній директорії та повертає його унікальний ідентифікатор.
load_in завантажує запис за його ідентифікатором із вказаної директорії.
update_in зливає наданий патч у існуючий запис та оновлює його час останньої модифікації.
list_in отримує список усіх записів у вказаній директорії, відсортованих за часом створення у порядку спадання.

## Публічний API

create_in — Створює новий запис у `dir` зі статусом `pending` та повертає його ідентифікатор.
load_in — Знаходить і повертає запис у `dir` за його ідентифікатором.
update_in — Оновлює запис, об'єднуючи дані з `patch` та збільшуючи час останньої зміни.
list_in — Виводить список усіх записів у `dir`, сортуючи їх від найновішого до найстарішого за датою створення.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
