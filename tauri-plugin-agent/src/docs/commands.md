# commands.rs

## Огляд

Модуль надає команди, експортовані як `plugin:agent|<command>`, для взаємодії з журналом подій. Ці команди дозволяють створювати, завантажувати, оновлювати та переглядати записи. Команди журналювання визначають шлях до каталогу запитів для кожного застосунку, виходячи з локального каталогу даних застосунку. Модуль також надає функціонал для зчитування конфігурації, що визначається `omlx_config`, яка спирається на файл settings.json. У разі виникнення помилок, модуль реалізує fail-safe механізм, повертаючи порожнє значення замість викидання винятків.

## Поведінка

journal_create створює запис у журналі, використовуючи вказаний намір та актора.
journal_load отримує запис з журналу за його ідентифікатором.
journal_update оновлює існуючий запис у журналі за його ідентифікатором, застосовуючи патч.
journal_list отримує список усіх записів у журналі.
OmlxConfig описує структуру конфігурації, що містить базовий URL та ключ API для omlx.
omlx_config зчитує базовий URL та ключ API з файлу `~/.omlx/settings.json`.

## Публічний API

journal_create — створює новий журнал для записів.
journal_load — завантажує існуючий журнал у пам'ять.
journal_update — змінює вміст вже існуючого журналу.
journal_list — відображає список доступних журналів.
OmlxConfig — зберігає базовий URL та ключ API для конфігурації Omlx.
omlx_config — надає налаштування конфігурації Omlx.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
