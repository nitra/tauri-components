# mcp_bridge.rs

## Огляд

Модуль створює loopback MCP HTTP сервер для виставлення каталогу інструментів застосунку. Каталог реєструється один раз за допомогою `acp_register_catalog` і дозволяє іншим компонентам системи, зокрема агенту ACP, знайти його у списку `mcpServers` при запуску. Сервер працює на адресі, подібній до http://127.0.0.1:54321/. При виклику інструментів через `tools/call` цей мост не виконує перевірку схеми чи tier-gating; він лише генерує подію `acp://mcp-tool-call` і очікує на відповідь через `acp_mcp_tool_result`. Модуль забезпечує стійкість, перехоплюючи помилки (fail-safe) та не генеруючи винятків назовні.

## Поведінка

McpBridgeState зберігає каталог доступних інструментів та відстежує очікування відповідей на виклики інструментів.
acp_register_catalog реєструє каталог інструментів, наданий застосунком.
acp_mcp_tool_result розв'язує очікування виклику інструменту, надсилаючи результат від вебв'ю.
acp_start_mcp_bridge запускає локальний HTTP-сервер для доступу до каталогу інструментів, повертаючи його базовий URL, наприклад, http://127.0.0.1:54321/.

## Публічний API

McpBridgeState — Зберігає зареєстрований каталог (з назвами, короткими описами та вхідними даними) та очікувані відповіді на виклики інструментів, прив'язані до ідентифікатора запиту.
acp_register_catalog — Замінює зареєстрований каталог, що виконується при ініціалізації агента або при зміні каталогу програми.
acp_mcp_tool_result — Завершує очікування відповіді на виклик інструменту, використовуючи обгортку диспетчера веб-перегляду.
acp_start_mcp_bridge — Запускає внутрішній HTTP-сервер MCP для програми та повертає його базовий URL, наприклад, http://127.0.0.1:54321/, для передачі в конфігурацію агентів.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
