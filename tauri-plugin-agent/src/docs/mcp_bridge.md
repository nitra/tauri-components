## Огляд

Цей модуль виставляє каталог інструментів застосунку, зареєстрований через `acp_register_catalog`, як loopback MCP HTTP server. Це дозволяє агенту MCP, запущеному через `acp::acp_spawn_agent`, знайти цей каталог у списку `mcpServers`. Сервер доступний за адресою, наприклад, <http://127.0.0.1:54321/>. При обробці викликів інструментів через `tools/call` міст не виконує перевірку схеми чи tier-gating; натомість він емітує подію `acp://mcp-tool-call` і очікує на виклик `acp_mcp_tool_result`. Міст перехоплює помилки (fail-safe) і не кидає винятків назовні, повертаючи порожнє значення (наприклад, null) у разі певних збоїв.

## Поведінка

McpBridgeState зберігає зареєстрований каталог інструментів та відстежує очікувані відповіді на виклики інструментів.
acp_register_catalog реєструє каталог інструментів, наданий з зовнішнього джерела, у внутрішній стан.
acp_mcp_tool_result розв'язує очікування виклику інструменту, надсилаючи відповідь від вебв'ю до відповідного запиту.
acp_start_mcp_bridge запускає локальний HTTP-сервер для доступу до каталогу інструментів і повертає його базовий URL, наприклад, <http://127.0.0.1:54321/>.

## Публічний API

McpBridgeState — Зберігає зареєстрований каталог (з інформацією про назву, опис, вхідні дані) та очікувані відповіді на виклики інструментів, використовуючи унікальний ідентифікатор запиту.
acp_register_catalog — Замінює зареєстрований каталог, що відбувається при ініціалізації агента або при зміні каталогу програми.
acp_mcp_tool_result — Обробляє відповідь на очікуваний виклик інструменту, надсилаючи її через механізм диспетчеризації веб-перегляду.
acp_start_mcp_bridge — Запускає внутрішній HTTP-сервер для зв'язку з MCP для цієї програми та повертає його базовий URL, наприклад, <http://127.0.0.1:54321/>, для передачі в конфігурацію агентів.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
